### §1.5 技データ参照ルール (SSOT: data/moves.json)

> **★ 原則**: 技データ (威力 / 命中 / 分類 / タイプ / 追加効果) は **構築ファイル内に複製しない**。
> SSOT は `~/.claude/skills/pokemon-champions/data/moves.json` (Showdown commit pinned in `data/VERSION.json`)。
> ダメ計バイナリ (`bin/pokechamp-calc`) と同じ原則 = データ二重管理を排除。

#### lookup 方法 (1 行で実行)

```bash
python3 ~/.claude/skills/pokemon-champions/scripts/lookup_move.py <技名>
# 例: python3 scripts/lookup_move.py ポルターガイスト しんそく ヘドロウェーブ
# JP/EN 両対応。威力 / 命中 / 分類 / タイプ / 優先度 / 追加効果を表示
```

#### 構築ファイル記述ルール

- **§1 構築一覧**: 技名のみ列挙 (4 技、ハイフン区切り)
- **§2 個別マニュアル「技 N 採用根拠」**: データ値 (威力/命中/分類等) は記述してよいが **moves.json から lookup した値であること** を明記、推測値は禁止
- **§3 TOP100 担当一覧**: 与/被ダメは確定数のみ。技仕様は moves.json 参照

#### 鉄則

- ★ 推測禁止: 不明な技仕様は必ず lookup
- ★ コピー禁止: moves.json の値を構築ファイルに転記する場合「(moves.json: poltergeist より)」等の出典明記
- ★ 更新検知: VERSION.json の `showdown_commit_date` が古ければ `bun scripts/extract_data.ts` で再生成

```
[こおり]
  - 解除確率 20%/T → 平均 5T
  - こごえるかぜ / 冷凍ビーム の凍結率 10% (相手対応で警戒)

[どく / もうどく]
  - Champions 仕様: 1/8 → 累積 1/16, 2/16, 3/16... (もうどく)
  - 8T で死ぬ前提の削り戦 (本構築の主軸戦術)
  - こちら どくどく被弾: 受けポケ (カバ / ブラ) は致命的、即引き
```

### 8.7 運要素のリカバリー指針

```
[急所 (確率 1/24、約 4%)]
  - 受けポケが急所被弾 → 受け崩される、即引き判断
  - こちら急所引き → ボーナス、押し切る

[命中率管理]
  - ヘドロウェーブ 100% / シャドボ 100% / イカサマ 100% (安定)
  - がんせきふうじ 95% / こごえる 95% / 大文字 (相手) 85% (やや不安)
  - きあいだま (採用してれば) 70% (要保険)
  - がんふう 5% 外し: 連打で確率低下 (3 回打って 1 回外す確率 14%)、外したら次でケア

[フリンチ]
  - 相手 ねこだまし: フリンチ 100% → ブラ せいしんりょくで無効
  - 相手 アイアンヘッド (ハッサム等): フリンチ 30% → ガブ がんふう S-1 で先制取り
  - 相手 エアスラ (メガカイ等): フリンチ 30% → こちら速度勝ちなら無関係

[凍結誘発]
  - こちら こごえる / メガカイ 冷ビ採用してないので凍結期待値ゼロ
  - 相手の凍結技 (マンムー こごえる等): 警戒のみ

[追加効果期待値計算]
  - シャドボ D-1 20% × 5 連打 = D-1 入る確率 67%
  - ヘドロウェーブ どく 10% × 10 連打 = どく入る確率 65%
  - これらを「保険として打つ価値があるか」で判断
  - ★ 受けポケ (HP 多い) ほど連打回数稼げる = 追加効果期待値高い
```
